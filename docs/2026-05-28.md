## 패키지 구조 확정

```
src/
├── domain/                        # 무변경 (엔티티, 비즈니스 로직)
├── controller/
│   ├── cli/                       # 기존 UI → 이동 (LoginController, CS01~CL04 등)
│   │   ├── customer/
│   │   └── employee/
│   └── web/                       # 신규 REST 컨트롤러
│       ├── AuthController.java
│       ├── ProductController.java
│       ├── SubscriptionController.java
│       ├── AccidentController.java
│       ├── ClaimController.java
│       └── dto/                   # Web 전용 Request/Response
├── common/
│   ├── auth/                      # (예정) UserSession 공통 인터페이스
│   └── exception/
│       ├── domain/                # DomainException, ValidationException, InvalidStatusTransitionException
│       ├── dao/                   # DataAccessException, DatabaseException
│       └── infra/                 # InfraException, UnauthorizedException, ForbiddenException, PageOutOfRangeException
└── infra/
    ├── web/                       # HTTP 기술 인프라 (Server, DispatcherServlet, Router, JWT)
    │   ├── auth/                  # JwtFilter, JwtUtil, WhiteList
    │   └── dto/                   # HttpRequest, HttpResponse, PageRequest, PageResponse
    ├── config/                    # AppConfig, DBConfig, JwtConfig, ServerConfig, ConfigLoader
    ├── persistence/               # Database (커넥션 풀), SqlExecutor, ResultSetExtractor
    ├── dao/                       # 각 도메인 DAO
    └── external/                  # 외부 연동 클라이언트
```

## 아키텍처 결정 사항

### 컨트롤러 분리 전략
- `controller/web/` vs `controller/cli/` — transport 종류 기준으로 최상위 분류
- 도메인별 서브패키지 없이 플랫하게 유지 (리버스 엔지니어링 가독성)
- CLI 클래스는 Scanner + 도메인 호출 혼재 → 그대로 유지 (CLI = controller + view 통합)
- Web 컨트롤러는 HTTP → 도메인 호출 → HTTP 순수 매핑

### 예외 계층
```
common/exception/
  domain/    DomainException (abstract) → ValidationException, InvalidStatusTransitionException
  dao/       DataAccessException (abstract) → DatabaseException
  infra/     InfraException (abstract) → UnauthorizedException(401), ForbiddenException(403), PageOutOfRangeException(400)
```
- `ServiceException` 미사용 — domain/infra/dao 3계층으로 정리
- `DispatcherServlet`에서 DomainException → 400, DataAccessException → 500, InfraException → 각 status

### DTO 위치
- `controller/web/dto/` — Web 전용, 도메인까지 내려가지 않음
- CLI는 DTO 없음 (Scanner 직접 파싱)

### 인증
- JWT (jjwt 라이브러리) — `JwtUtil`에서 access 토큰 발급
- `JwtFilter` — `Authorization: Bearer <token>` 검증, WhiteList(/auth/login) 제외
- 로그인 성공 시 `AuthResponse(accessToken, name, role)` 반환

### 실행 모드
- `java -jar app.jar web` → HTTP 서버 시작 (기본값)
- `java -jar app.jar cli` → 기존 CLI 실행
- `AppContext.initialize(AppConfig)` 에서 CLI/Web 공통 조립

## 신규 파일 목록

### infra/web (HTTP 기술 인프라)
| 파일 | 역할 |
|---|---|
| `Server.java` | HttpServer 래퍼, 스레드풀 5 |
| `DispatcherServlet.java` | Front Controller, 예외 핸들링 |
| `Router.java` | 메서드+경로 패턴 라우팅, path variable 지원 |
| `RouteHandler.java` | `@FunctionalInterface` 핸들러 |
| `auth/JwtFilter.java` | Bearer 토큰 검증 |
| `auth/JwtUtil.java` | 토큰 발급/검증 (HMAC-SHA) |
| `auth/WhiteList.java` | 인증 제외 경로 |
| `dto/HttpRequest.java` | 요청 래퍼 (path var, query param, body 역직렬화) |
| `dto/HttpResponse.java` | 응답 래퍼 (ok/created/noContent/error) |
| `dto/ErrorBody.java` | 에러 응답 포맷 `{status, message}` |
| `dto/PageRequest.java` | page/limit 쿼리 파라미터 파싱 |
| `dto/PageResponse.java` | 페이지네이션 응답 포맷 |

### infra/config
| 파일 | 역할 |
|---|---|
| `AppConfig.java` | DB/JWT/Server 설정 조립 |
| `ConfigLoader.java` | 설정 로딩 인터페이스 |
| `PropertiesConfigLoader.java` | .properties 구현체 |
| `ServerConfig.java` | 바인딩 주소 |
| `JwtConfig.java` | secret, 만료 시간 |

### common/exception
| 파일 | 역할 |
|---|---|
| `dao/DataAccessException.java` | DAO 예외 추상 클래스 (status 포함) |
| `infra/InfraException.java` | 인프라 예외 추상 클래스 |
| `infra/UnauthorizedException.java` | 401 |
| `infra/ForbiddenException.java` | 403 |
| `infra/PageOutOfRangeException.java` | 400 페이지 범위 초과 |

### controller/web
| 파일 | 엔드포인트 | 대응 use case |
|---|---|---|
| `AuthController.java` | POST /auth/login | 로그인 |
| `ProductController.java` | GET /products, GET /products/{id}, POST /products, PUT /products/{id}/approval·rate-verification·sale, POST /products/{id}/estimate | CS-02, CS-03, CT-01, CT-04~06 |
| `SubscriptionController.java` | GET /subscriptions, GET /subscriptions/{no}, POST /subscriptions, GET /subscriptions/pending, PUT /subscriptions/{no}/review | CS-01, CS-05, UW-01 |
| `AccidentController.java` | GET /accidents, GET /accidents/{id}, POST /accidents, PUT /accidents/{id}/assign | CS-04, CL-01 |
| `ClaimController.java` | GET /claims, GET /claims/{id}, PUT /claims/{id}/assess, PUT /claims/{id}/pay | CL-02, CL-04 |

### controller/web/dto
`LoginRequest`, `AuthResponse`, `ProductCreateRequest`, `ProductResponse`, `PremiumEstimateRequest`, `PremiumEstimateResponse`, `SubscriptionCreateRequest`, `SubscriptionResponse`, `AccidentReportRequest`, `AccidentResponse`, `ClaimAssignRequest`, `ClaimResponse`

### 기타
- `resources/application.properties` — DB/JWT/서버 설정 파일
- `Main.java` — 모드 분기 (web/cli)
- `infra/AppContext.java` — CLI + Web 공통 조립
